# 🎮 HeartQuest_POO - Guía Básica para el Equipo

¡Hola equipo! Este es el repositorio de nuestro juego. Si nunca has usado GitHub o Unity, no te asustes. Esta guía está diseñada paso a paso para que puedas descargar el juego, abrirlo en tu computadora y subir tus cambios sin romper el trabajo de los demás. 

Por favor, lee TODO con calma y sigue las instrucciones al pie de la letra.

---

## 🛠️ ¿Qué necesitas instalar primero?
Antes de descargar el juego, necesitas tener instalados estos tres programas:

1. **Unity Hub**: Es el administrador de Unity. [Descárgalo aquí](https://unity.com/download).
2. **Unity Versión 6000.4.0f1**: Una vez dentro de Unity Hub, ve a *Installs*, luego a *Install Editor*, busca esta versión e instálala (incluye el soporte para el editor de código, como Visual Studio o VS Code). Si usas una versión diferente, el proyecto nos va a fallar.
3. **GitHub Desktop**: [Descárgalo aquí](https://desktop.github.com/). Es una aplicación muy fácil y visual para subir y descargar el juego sin tener que tocar códigos raros en una terminal negra.

---

## 📥 Paso 1: Cómo descargar el juego (Clonación)
No se debe descargar como un `.zip`. Tenemos que "clonarlo" para que podamos mandar nuestras actualizaciones y mantener el historial.

1. Abre **GitHub Desktop**.
2. Te pedirá iniciar sesión. Usa tu cuenta de GitHub.
3. Ve al menú superior y selecciona: `File` -> `Clone repository...` (Clonar repositorio).
4. Ve a la pestaña que dice **URL** y pega este link: `https://github.com/is0gvs/HeartQuest_POO.git`
5. En **Local path** (Ruta local), elige una carpeta en tu computadora donde quieras guardar el juego (por ejemplo: `Mis Documentos/ProyectosU`). *No lo guardes en Descargas*.
6. Haz clic en el botón azul **Clone**. ¡Espera unos segundos a que termine!

---

## 🕹️ Paso 2: Cómo abrir el juego en Unity
1. Abre **Unity Hub**.
2. Dale al botón arriba a la derecha que dice **Add** (Agregar) o **Open** (Abrir).
3. Busca la carpeta que acabas de clonar (probablemente se llame `HeartQuest_POO` o `AntiBullying`). 
4. **IMPORTANTE:** Selecciona esa carpeta principal contenedora y dale a "Seleccionar Carpeta" (no te metas dentro de las subcarpetas *Assets* o *Library*).
5. El proyecto te aparecerá ahora en la lista inicial. Asegúrate de que a la derecha dice que usa la versión `6000.4.0f1`.
6. Haz clic sobre el proyecto para abrirlo. La primera vez se puede demorar unos cuantos minutos configurando todo. ¡Ten paciencia!

---

## 🤝 Paso 3: Cómo trabajar en equipo (La regla de oro)

Hay un ciclo de trabajo vital que no debemos romper para no perder el progreso de nadie. Haz de cuenta que estamos editando un documento compartido, pero de forma manual.

### ⚠️ ANTES de empezar a trabajar (Cuando te sientas en el PC):
¡Siempre revisa si alguien más hizo progreso mientras tú no estabas!
1. Abre **GitHub Desktop**.
2. Haz clic en botón de arriba que dice **Fetch origin** (o *Pull*). Esto descargará automáticamente los últimos cambios que hicieron los demás a tu computadora.
3. Una vez descargado, ahora sí, abre Unity y ponte a trabajar.

### ✅ CUANDO TERMINES de trabajar (Para compartir tus cambios):
Terminaste por hoy, o terminaste una tarea pequeña. Es hora de mandar tus cambios:
1. **¡Guarda tu escena en Unity!** (Menú *File* -> *Save*, o presionando `Ctrl+S`).
2. Ve a **GitHub Desktop**. A la izquierda verás una lista verde/amarilla con todos los archivos que acabas de modificar.
3. Abajo a la izquierda hay un campito que dice *Summary* (Resumen). **Escribe ahí qué fue lo que hiciste.** ¡Sé claro para que los demás entiendan! *Ejemplo: "Decoré el nivel 1 de la escuela" o "Agregué el sonido de salto"*.
4. Dale al botón azul de abajo que dice **Commit to main**. (Esto guarda tus cambios en tu propio computador como un "paquete").
5. Finalmente, dale al botón azul grandote de arriba que dice **Push origin** (Empujar). Esto sube el "paquete" a la nube para que esté disponible para todo el equipo.

¡Y Listo! Si te acostumbras a hacer este proceso (Descargar -> Trabajar -> Guardar -> Commit -> Push) el proyecto va a salir perfecto y no vamos a perder código.
